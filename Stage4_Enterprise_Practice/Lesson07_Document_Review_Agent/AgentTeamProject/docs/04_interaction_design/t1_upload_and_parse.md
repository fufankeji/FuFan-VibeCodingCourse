# 交互链路规范：上传入口 → 文档校验 → 解析任务创建

**文档编号**：04_interaction_design/t1_upload_and_parse
**编写日期**：2026-03-11
**负责范围**：Teammate 1（上传入口、文档校验、解析任务创建）
**输入文档**：`03_problem_modeling/problem_model.md`
**关联实体**：`Contract`、`ReviewSession`（state: parsing）
**不涉及范围**：风险扫描（scanning）、HITL 审批（hitl_pending）、报告生成

---

## 一、阶段概述

本文档覆盖合同审核系统用户旅程的**第一阶段**，即从用户触达上传入口到系统完成文档解析、ReviewSession 进入下一阶段（scanning）为止的完整交互链路。

本阶段包含三个顺序子阶段：

| 子阶段 | 描述 | 系统边界 |
|--------|------|----------|
| **1. 文件上传** | 用户通过界面选取合同文件并上传至系统 | 文件传输完成，服务端持久化文件 |
| **2. 文档校验** | 系统对上传文件进行格式、大小、内容可读性的自动校验 | 校验通过后创建 Contract 记录 |
| **3. 解析任务创建与执行** | 系统调用外部 OCR（如达观 TextIn）完成文本解析，ReviewSession.state = parsing | 解析完成，state 流转至 scanning |

**设计原则**：
- 操作全程可追溯，每个关键动作记录操作人、时间戳、操作类型（对应设计红线：操作全程可追溯）
- OCR 解析不自研，接入外部方案（如达观 TextIn），系统负责任务调度与状态管理
- 低置信度内容（OCR 解析质量不足时）在后续阶段必须以醒目标识提示人工核对（本阶段完成检测，后续阶段执行标记）
- 合同文件不得明文传输至不受控的外部 API，OCR 接口调用须符合数据安全约束

---

## 二、用户角色与入口

### 2.1 涉及角色

| 角色 | role 枚举值 | 本阶段操作 |
|------|-------------|-----------|
| 法务合同审核人员 | `reviewer` | 上传合同、查看校验结果、查看解析进度、处理解析失败 |
| 业务方/合同发起人 | `submitter` | 上传合同、查看解析进度（只读） |

### 2.2 系统入口点

用户从以下两个入口进入本阶段：

1. **主导航入口**：首页或工作台的「新建审核」按钮，点击后跳转至上传页
2. **合同列表入口**：合同列表页右上角的「上传合同」按钮，点击后打开上传面板（可为侧边抽屉或全屏弹窗，原型规范阶段决定）

## 四、各状态的界面规范

### 状态 S0：上传初始态（Idle）

**触发条件**：用户进入上传页面且尚未选取文件。

**界面呈现**：
- 上传区域：矩形虚线边框区域，居中展示「拖拽文件至此处」提示文字
- 区域内展示支持格式说明：「支持 PDF、DOCX 格式，文件大小不超过 50MB」
- 区域下方有「点击选取文件」按钮（作为拖拽的备选操作方式）
- 页面顶部展示当前用户信息（上传人将被记录为 Contract.uploaded_by）

**用户操作**：
- 将文件拖入虚线区域，或点击「点击选取文件」按钮打开系统文件选取对话框

**系统响应**：
- 拖拽悬停时：上传区域高亮（边框颜色加深，背景色变化），提示文字切换为「释放以选取文件」
- 文件选取后：进入前端校验（S1）

---

### 状态 S1：前端格式预校验（Client Validation）

**触发条件**：用户完成拖拽释放或在文件选取对话框中确认文件。

**校验内容（前端执行）**：

| 校验项 | 规则 | 执行位置 |
|--------|------|----------|
| 文件扩展名 | 必须为 `.pdf` 或 `.docx` | 前端 |
| MIME 类型 | 必须为 `application/pdf` 或 `application/vnd.openxmlformats-officedocument.wordprocessingml.document` | 前端 |
| 文件大小 | 不超过 50MB（52,428,800 字节） | 前端 |
| 单次上传数量 | 仅允许选取 1 个文件（禁止批量上传） | 前端 |

**界面呈现**：
- 校验过程在本地即时完成，无需等待状态，用户感知不到此状态存在
- 若校验通过，无缝进入 S2（上传中）
- 若校验不通过，进入 S_ERR_1（前端校验失败）

**用户操作**：无（系统自动执行）

**系统响应**：校验通过则立即开始文件上传；校验不通过则展示错误提示。

---

### 状态 S2：上传中（Uploading）

**触发条件**：前端校验通过，开始向服务端传输文件。

**界面呈现**：
- 上传区域内展示文件名、文件大小
- 进度条展示上传百分比（0% → 100%）
- 进度条下方展示「上传中，请勿关闭页面」提示文字
- 「点击选取文件」按钮变为不可用状态（disabled）
- 提供「取消上传」操作入口（小字链接或次要按钮）

**用户操作**：
- 等待上传完成（推荐操作）
- 点击「取消上传」：中断传输，返回 S0（初始态），文件记录不创建

**系统响应**：
- 通过 HTTP multipart/form-data 分块上传（具体协议由 API 规范定义）
- 每个分块上传完成后更新前端进度百分比
- 上传完成后进入服务端校验（S3）
- 若网络中断超过 30 秒无响应，自动切换至 S_ERR_2（上传失败）

---

### 状态 S3：服务端校验中（Server Validation）

**触发条件**：文件传输 100% 完成，服务端开始执行校验。

**界面呈现**：
- 进度条保持在 100%，进度条状态变为「校验中...」
- 展示文件名和文件大小（与 S2 保持一致）
- 用户无可操作项，界面处于只读等待状态

**校验内容（服务端执行）**：

| 校验项 | 规则 | 失败进入状态 |
|--------|------|------------|
| 文件格式二次验证 | 解析文件头（Magic Bytes），确认与声称类型一致 | S_ERR_3 |
| 文件大小二次验证 | 服务端再次确认不超过 50MB | S_ERR_3 |
| 文件完整性 | 校验文件未损坏（PDF 结构完整，DOCX 可解压） | S_ERR_3 |
| 内容可读性预检 | 检测文件是否包含可提取的文本层（区分扫描件 vs 数字原生 PDF） | S_WARN_1（可读性警告，不阻断） |
| 文件加密检测 | 检测 PDF 是否有密码保护或内容加密 | S_ERR_3 |
| 空文档检测 | 检测页数是否为 0 或正文字符数为 0 | S_ERR_3 |

**说明（内容可读性预检）**：
- 扫描件 PDF（无文本层）可通过校验但进入 S_WARN_1，系统提示「检测为扫描件，将依赖 OCR 识别，提取置信度可能偏低」
- 数字原生 PDF/DOCX（含文本层）直接通过，无警告
- 本项检测结果传递给解析阶段（S6），影响 ExtractedField.needs_human_verification 的初始判断

**用户操作**：无（等待系统响应）

**系统响应**：
- 校验通过（含 S_WARN_1）：在数据库创建 Contract 记录（状态为已上传），进入 S4（创建审核会话）
- 校验不通过：进入 S_ERR_3（服务端校验失败）

---

### 状态 S4：创建审核会话（Session Initialization）

**触发条件**：服务端校验通过。

**界面呈现**：
- 展示「正在初始化审核任务...」文字，附带加载动画
- 界面无用户可操作项

**系统响应（后台）**：
1. 创建 `Contract` 记录，写入 `file_path`、`file_type`、`uploaded_by`、`uploaded_at`
2. 创建 `ReviewSession` 记录，`state = parsing`，生成 `langgraph_thread_id`
3. 记录操作日志（操作人、时间戳、操作类型：contract_created）
4. 触发 OCR 解析任务，进入 S5（解析中）

**状态流转**：Contract 创建 → ReviewSession.state 设置为 `parsing` → 进入 S5

---

### 状态 S5：解析中（Parsing）

**触发条件**：ReviewSession.state = `parsing`，OCR 解析任务已提交至外部服务（如达观 TextIn）。

**界面呈现**：
- 页面切换为「解析进度页面」（或「审核任务详情页」的解析状态视图）
- 展示区域包含：
  - **文件信息卡片**：文件名、上传时间、上传人
  - **解析进度指示器**：
    - 文字说明：「正在解析合同文档...」
    - 视觉进度：不确定性进度条（indeterminate）或阶段性步骤指示器
    - 预计时间提示：「预计需要 1-3 分钟，请耐心等待」（固定文案，不实时推算）
  - **操作说明**：「解析期间可离开此页面，完成后将通过站内通知提醒您」
  - **放弃任务**入口：次要操作按钮「放弃本次审核」（进入 S_CANCEL 确认流程）

**进度感知设计说明**：
- OCR 外部服务不提供细粒度进度回调时，前端展示不确定性进度条（非百分比），避免进度误导
- 若外部服务支持进度回调，可展示阶段描述（如「页面识别中 3/10 页」）
- 系统每 10 秒轮询一次解析任务状态（或通过 WebSocket/SSE 接收推送），超时判定见 S_ERR_5

**用户操作**：
- 保持页面等待：推荐操作
- 离开页面：允许，系统在后台继续执行；用户通过通知中心或合同列表查看最新状态
- 点击「放弃本次审核」：弹出确认对话框（见 S_CANCEL）

**系统响应**：
- 解析成功：进入 S6（解析完成）
- 解析超时（超过 15 分钟无结果）：进入 S_ERR_5（解析超时）
- 解析服务返回错误：进入 S_ERR_4（解析失败）

---

### 状态 S6：解析完成（Parse Succeeded）

**触发条件**：外部 OCR 服务返回解析结果，系统完成文本结构化写入。

**界面呈现**：
- 简短过渡提示：「解析完成，正在进入风险扫描...」（展示约 1.5 秒）
- 无用户交互要求

**系统响应（后台）**：
1. 将 OCR 输出写入数据库（关联至 ReviewSession 的文本存储）
2. 对每个提取字段生成置信度评分（来源：OCR 服务返回的置信度 + 内容可读性预检结果综合判断）
3. confidence_score 低于阈值（参考值：< 70）的字段，设置 `ExtractedField.needs_human_verification = true`
4. 记录操作日志（操作类型：parse_completed，包含 OCR 服务版本、耗时）
5. 将 `ReviewSession.state` 由 `parsing` 更新为 `scanning`
6. **自动跳转**：页面自动重定向至「风险扫描阶段」（Teammate 2 负责范围）

**状态流转**：`ReviewSession.state: parsing → scanning`

---

### 状态 S_WARN_1：扫描件可读性警告（Scanned Document Warning）

**触发条件**：服务端内容可读性预检检测到文件为扫描件（无文本层 PDF）。

**界面呈现**：
- 在 S3 完成后、进入 S4 之前，展示一个警告横幅（非阻断性，可关闭）
- 警告文案：「检测到此文件为扫描版 PDF，将依赖 OCR 识别进行文本提取。识别结果可能存在误差，低置信度字段将在审核阶段标记供人工核对。」
- 提供两个操作按钮：
  - 「继续提交」（主要操作）：关闭警告，继续进入 S4
  - 「重新上传」（次要操作）：返回 S0，允许用户上传更高质量版本

**系统响应**：
- 用户选择「继续提交」：进入 S4，同时在 ReviewSession 元数据中标记 `is_scanned_document = true`
- 用户选择「重新上传」：已上传的临时文件清理，返回 S0

---

### 状态 S_CANCEL：放弃确认（Abort Confirmation）

**触发条件**：用户在 S5（解析中）点击「放弃本次审核」。

**界面呈现**：
- 弹出确认对话框（Modal）
- 标题：「确认放弃本次审核？」
- 内容：「放弃后，已上传的合同文件及当前解析进度将被清除，此操作不可撤销。」
- 操作按钮：
  - 「取消」：关闭对话框，返回 S5 继续等待
  - 「确认放弃」（危险操作样式，红色或高对比度）：执行放弃

**系统响应（确认放弃后）**：
1. 向 OCR 服务发送取消请求（如服务支持）
2. 将 `ReviewSession.state` 更新为 `aborted`
3. 记录操作日志（操作类型：session_aborted，记录操作人和时间戳）
4. 页面跳转至合同列表或工作台首页
5. 已上传文件按数据保留策略处理（具体策略由架构设计阶段定义）

## 五、错误处理规范

### S_ERR_1：前端校验失败

**触发条件**：文件扩展名、MIME 类型、文件大小、文件数量不符合规则（前端检测）。

**错误类型与提示文案**：

| 错误原因 | 提示文案 | 引导操作 |
|----------|----------|----------|
| 文件格式不支持 | 「不支持此文件格式，请上传 PDF 或 DOCX 格式的合同文件」 | 重新选取文件 |
| 文件大小超限 | 「文件大小超过 50MB 限制，当前文件为 {X}MB，请压缩后重新上传」 | 重新选取文件 |
| 选取多个文件 | 「每次只能上传一份合同文件，请重新选取」 | 重新选取文件 |

**界面行为**：
- 错误提示显示在上传区域下方（或以 Toast 形式展示），颜色使用错误色（红色系）
- 文件选取状态重置，上传区域恢复至 S0 初始态
- 错误提示在用户重新操作后自动消失

---

### S_ERR_2：上传失败（网络/服务端 5xx）

**触发条件**：上传过程中网络中断、请求超时（> 30 秒无响应）、或服务端返回 5xx 错误。

**界面呈现**：
- 进度条展示为失败状态（颜色变为红色，进度停止）
- 提示文案：「上传失败，{失败原因}」，失败原因映射如下：

| 错误码 | 展示文案 |
|--------|----------|
| 网络超时 | 「上传失败，网络连接超时，请检查网络后重试」 |
| 服务端 5xx | 「上传失败，服务器暂时不可用，请稍后重试」 |
| 其他未知错误 | 「上传失败，请稍后重试（错误码：{code}）」 |

**用户操作**：
- 「重试上传」按钮：重新发起上传，进入 S2
- 「取消」按钮：返回 S0 初始态

---

### S_ERR_3：服务端校验失败

**触发条件**：服务端校验发现文件格式伪造、文件损坏、文件加密、空文档等问题。

**界面呈现**：
- 上传区域展示失败结果（文件名 + 错误状态图标）
- 错误提示文案：

| 错误原因 | 展示文案 | 用户引导 |
|----------|----------|----------|
| 文件损坏 | 「文件无法读取，可能已损坏，请检查文件后重新上传」 | 重新上传 |
| 格式伪造（扩展名与实际类型不符） | 「文件格式验证失败，请确认文件未被重命名，上传原始格式的 PDF 或 DOCX 文件」 | 重新上传 |
| 文件已加密 | 「检测到 PDF 文件已加密，请移除密码保护后重新上传」 | 移除加密后重新上传 |
| 空文档 | 「文件内容为空或无法读取正文，请确认文件内容后重新上传」 | 重新上传 |

**用户操作**：
- 「重新上传」按钮：返回 S0 初始态
- 错误信息中提供简短的操作指引（如「如何移除 PDF 密码保护」链接，原型规范阶段确认是否纳入 MVP）

---

### S_ERR_4：解析失败（OCR 服务错误）

**触发条件**：外部 OCR 服务（如达观 TextIn）返回错误响应（非超时）。

**界面呈现**：
- 解析进度页面切换为失败状态
- 展示文案：「文档解析失败，{失败原因描述}」
- 提供以下操作入口：
  - 「重新解析」按钮（主要操作）：重新提交 OCR 解析任务，进入 S5
  - 「查看错误详情」展开项（次要操作）：展示原始错误信息（技术错误码 + 描述），供用户截图反馈
  - 「放弃本次审核」链接（危险操作）：进入 S_CANCEL 确认流程

**重试限制**：同一 ReviewSession 最多允许手动重试 3 次，超出后「重新解析」按钮置灰，展示提示「已达最大重试次数，请联系系统管理员」。

**日志记录**：每次解析失败记录操作日志（操作类型：parse_failed，包含 OCR 服务返回的错误码和时间戳）。

---

### S_ERR_5：解析超时

**触发条件**：解析任务提交后超过 15 分钟未收到 OCR 服务结果（包含用户离开页面后台轮询超时的场景）。

**界面呈现**：
- 与 S_ERR_4 界面一致，但错误文案为：「文档解析超时（超过 15 分钟未完成），可能因文件过大或服务繁忙导致」
- 操作入口与 S_ERR_4 相同（重新解析 / 查看错误详情 / 放弃本次审核）

**站内通知**：
- 若用户已离开页面，通知中心推送通知：「您于 {时间} 上传的合同《{合同名}》解析超时，请返回处理」，附带跳转链接

## 六、状态流转定义

### 6.1 ReviewSession.state 枚举（本阶段涉及部分）

本文档涉及以下 `ReviewSession.state` 值（完整枚举参见 `03_problem_modeling/problem_model.md` 领域模型）：

| 状态值 | 含义 | 进入条件 | 离开条件 |
|--------|------|----------|----------|
| `parsing` | OCR 解析任务已创建，正在执行中 | Contract 创建完成，OCR 任务提交成功 | 解析成功（→ scanning）或任务被放弃（→ aborted） |
| `aborted` | 审核会话已放弃 | 用户确认放弃，或系统检测到不可恢复的错误 | 终态，不可离开 |

### 6.3 操作日志事件类型枚举（本阶段产生）

| 事件类型（event_type） | 触发时机 | 关键附加字段 |
|------------------------|----------|-------------|
| `contract_uploaded` | 文件上传至服务端成功，服务端校验通过 | `file_name`, `file_size`, `file_type`, `is_scanned_document` |
| `contract_created` | Contract 数据库记录创建完成 | `contract_id`, `uploaded_by`, `uploaded_at` |
| `session_created` | ReviewSession 记录创建，state = parsing | `session_id`, `langgraph_thread_id` |
| `parse_started` | OCR 解析任务提交至外部服务 | `session_id`, `ocr_service_provider`, `submitted_at` |
| `parse_completed` | OCR 解析成功，结果已写入数据库 | `session_id`, `ocr_service_version`, `duration_seconds` |
| `parse_failed` | OCR 服务返回错误 | `session_id`, `error_code`, `error_message`, `retry_count` |
| `parse_timeout` | 解析任务超过超时阈值 | `session_id`, `elapsed_seconds`, `retry_count` |
| `session_aborted` | 用户确认放弃或系统触发 abort | `session_id`, `aborted_by`, `abort_reason` |

## 七、关键设计决策记录

| 决策点 | 决策内容 | 决策依据 |
|--------|----------|----------|
| 文件大小上限 | 50MB | 覆盖绝大多数企业合同（含长篇框架协议），同时控制 OCR 服务成本和解析时长 |
| 支持格式 | PDF + DOCX | 与 problem_model 场景 1 一致；DOC、TXT 等不在 MVP 范围 |
| 内容可读性预检位置 | 服务端校验阶段（S3）执行，非前端 | 前端无法可靠判断 PDF 文本层，需服务端解析文件结构 |
| 扫描件处理策略 | 不阻断上传，以警告引导 | 扫描件仍有处理价值，阻断会增加用户负担；通过低置信度标记保障质量 |
| 解析超时阈值 | 15 分钟 | 给外部 OCR 服务留足处理时间，同时避免用户无限期等待 |
| 手动重试上限 | 3 次 | 防止因系统故障导致无效重试消耗资源，超出后引导至管理员 |
| 解析进度展示 | 不确定性进度条（indeterminate） | OCR 外部服务通常不提供细粒度进度，模拟百分比会造成误导 |
| 自动跳转触发 | 解析成功后系统自动执行，无需用户确认 | 减少用户等待中的操作摩擦，与 problem_model 场景描述一致 |
